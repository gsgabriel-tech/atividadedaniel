A funcionalidade deste modelo se baseia em uma única ramificação nomeada “main” que usa duas ramificações para registrar o histórico do que foi feito, primeira etapa se serve para complementar a ramificação main padrão com uma ramificação develop. Um jeito simples de alcançar esse resultado é com um desenvolvedor criando uma ramificação develop no local e fazendo o push para o servidor, criando um git branch “develop” e um git push -u develop “ para enviar a alteração pro servidor, com esta ramificação vai se conter o histórico completo do projeto, enquanto a ramificação main vai conter uma versão abrevia.
Após isso executar “git flow init “no repositório existente para criar uma ramificação de desenvolvimento, cada novo recurso deve residir na própria ramificação, que pode ser enviada por push para o repositório central para backup/colaboração, quando um recurso é concluído, ele passa por merge de volta para a ramificação de desenvolvimento. Os recursos não devem nunca interagir direto com a ramificação main. É importante observar que as ramificações de recurso, combinadas com a ramificação de desenvolvimento são o fluxo de trabalho de ramificações de recursos, as mesmas , são em geral criadas a partir da ramificação de desenvolvimento mais recente, pode ser criar as ramificações de recurso com o comando “git flow feature start feature_branch” usando a extensão do gitflow. Ao concluir o trabalho de desenvolvimento, próximo passo é mesclar a ramificação de recurso de desenvolvimento, usando as extensões do gitflow, seria o respectivo comando: “git flow feature finish feature_branch”.
Quando a ramificação develop adquiriu recursos suficientes para um lançamento (ou uma data de lançamento predeterminada está se aproximando), você bifurca uma ramificação release a partir da ramificação develop. Criar esta ramificação dá início ao próximo ciclo de lançamento, portanto nenhum novo recurso pode ser adicionado depois deste ponto, apenas atualizações de segurança ou outras tarefas relacionadas ao lançamento, assim que estiver pronta para ser lançada, a ramificação release passa por merge para a ramificação main e é marcada com o número da versão.
A elaboração de ramificações de lançamento tambem é outra operação de ramificação simples, igual as ramificações de recurso, as ramificações de lançamento são baseadas na ramificação de desenvolvimento, para se criar uma nova ramificação de lançamento pode se usar o seguinte comando com a extensão do git-flow: "$ git flow release start 0.1.0". Depois que a versão estiver pronta para o lançamento, vai ser feito o merge dela na ramificação main e na ramificação develop e, então, a ramificação release vai ser excluída, e para finalizar a ramificação de lançamento, usa-se os seguinte comando: “ git flow release finish '0.1.0' ”.
Podemos destacar também as ramificações de hotfix , que são ramificações de manutenção,são usadas para corrigir com rapidez lançamentos de produção, a mesma pode ser criada usando o comando “ $ git flow hotfix start hotfix_branch “, parecido com a finalização da ramificação release, é feito um merge da ramificação hotfix na ramificação main e develop, exemplo: “$ git flow hotfix finish hotfix_branch”.
